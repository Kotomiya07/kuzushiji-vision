# くずし字ページ認識モデル - アーキテクチャ詳細

## 1. アーキテクチャ概要

このモデルは、古典籍の1ページ全体から文字を検出し、各文字のUnicodeを認識するEnd-to-Endのモデルです。

```
Input Page → Swin Transformer V2 → Feature Extraction → Transformer Encoder → Character Query Decoder → Positions & Unicode
```

## 2. 各コンポーネントの詳細

### 2.1 バックボーン (Swin Transformer V2)

- **目的**: 大規模画像からの階層的特徴抽出
- **構成**:
  - 階層的な構造により効率的な画像処理
  - ウィンドウベースの自己注意機構
  - パッチマージングによる特徴統合
- **利点**:
  - 大きな入力画像を効率的に処理
  - グローバルとローカルの文脈を捉える
  - 位置に依存しない特徴抽出

### 2.2 位置エンコーディング

- **2D位置エンコーディング**:
  - 画像の縦横方向の位置情報を個別に符号化
  - 学習可能なパラメータまたはサイン波ベース
  - 最大2048×2048ピクセルまでサポート

### 2.3 Transformerエンコーダー

- **目的**: 画像全体の文脈を考慮した特徴抽出
- **構成**:
  - マルチヘッド自己注意機構
  - 6層のエンコーダー層
  - 512次元の隠れ層
- **特徴**:
  - ページ全体の文脈を考慮
  - 文字間の関係性を捉える
  - 効率的な特徴統合

### 2.4 文字クエリデコーダー

- **目的**: 文字の位置とUnicodeの同時予測
- **構成**:
  - 学習可能な文字クエリ埋め込み
  - クロスアテンションによる特徴参照
  - 位置とUnicode予測ヘッド
- **特徴**:
  - 並列的な文字検出
  - End-to-Endの学習
  - 縦書きを考慮した出力順序

## 3. 入出力の詳細

### 3.1 入力処理

- **入力形式**: RGB画像（可変サイズ）
- **前処理**:
  ```python
  normalize_mean = [0.485, 0.456, 0.406]
  normalize_std = [0.229, 0.224, 0.225]
  ```

### 3.2 出力形式

- **文字位置**:
  - 正規化されたx, y座標（[0, 1]の範囲）
  - 文字の幅と高さ（[0, 1]の範囲）
  - 検出確信度スコア

- **文字認識**:
  - Unicode文字ID
  - 認識確信度スコア

### 3.3 文字の検出順序

縦書きの日本語文書に対応：
1. 右から左への走査
2. 各列内で上から下への走査
3. 確信度に基づくフィルタリング

## 4. 学習方法

### 4.1 損失関数

```python
total_loss = position_loss + unicode_loss

# 位置の損失（L1損失）
position_loss = L1Loss(predicted_pos, target_pos)

# Unicode予測の損失（クロスエントロピー）
unicode_loss = CrossEntropyLoss(predicted_unicode, target_unicode)
```

### 4.2 最適化設定

```python
optimizer = AdamW(
    lr=0.0001,
    weight_decay=0.0001,
    betas=[0.9, 0.999]
)

scheduler = CosineAnnealingLR(
    T_max=100,
    eta_min=0.000001
)
```

### 4.3 学習戦略

- バッチサイズ: 1（大きな入力画像のため）
- 勾配累積: 16ステップ
- 混合精度学習の使用
- 勾配クリッピング: 1.0

## 5. メモリと計算効率

- **入力サイズ**: 最大2048×2048ピクセル
- **GPU要件**: 16GB以上のVRAM推奨
- **バッチ処理**: 
  - 1画像ずつ処理
  - 勾配累積による疑似的な大バッチ
- **推論速度**: 約1秒/ページ（A100 GPU使用時）

## 6. 制限事項と改善点

### 6.1 現在の制限

- 1ページあたり最大1000文字まで検出可能
- メモリ使用量が大きい
