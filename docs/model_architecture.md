# くずし字認識モデル - アーキテクチャ詳細

## 1. アーキテクチャ概要

このモデルは、日本の古典籍に含まれるくずし字を認識するために設計された深層学習モデルです。画像特徴抽出とシーケンス生成を組み合わせたエンコーダー・デコーダー構造を採用しています。

```
Input Image → ResNet50 + FPN → Transformer Encoder → Transformer Decoder → Character Sequence
```

## 2. 各コンポーネントの詳細

### 2.1 バックボーン (ResNet50)

- **目的**: 画像から階層的な視覚特徴を抽出
- **構成**:
  - 事前学習済みResNet50を使用
  - 最初の2層は固定（転移学習の効果を高めるため）
  - 複数のスケールの特徴マップを出力（C3, C4, C5レイヤー）
- **出力サイズ**:
  - C3: 28×28×512
  - C4: 14×14×1024
  - C5: 7×7×2048

### 2.2 特徴ピラミッドネットワーク (FPN)

- **目的**: マルチスケール特徴の効果的な統合
- **構成**:
  - トップダウンパスによる高レベル特徴の伝播
  - ラテラル接続による低レベル特徴の保持
  - 3×3畳み込みによる特徴の洗練化
- **特徴**:
  - 各レベルで256チャネルに統一
  - スケール間の情報フローを確保
  - 文字の細部と全体構造の両方を捉える

### 2.3 Transformerエンコーダー

- **目的**: 文字の空間的関係性の学習
- **構成**:
  - マルチヘッドセルフアテンション (8ヘッド)
  - 6レイヤーのエンコーダー
  - 256次元の隠れ層
- **機能**:
  - グローバルな文脈の捕捉
  - 位置に依存しない特徴抽出
  - アテンションによる重要領域の強調

### 2.4 Transformerデコーダー

- **目的**: 文字シーケンスの生成
- **構成**:
  - マルチヘッドアテンション (8ヘッド)
  - 6レイヤーのデコーダー
  - クロスアテンションによるエンコーダー特徴の参照
- **特徴**:
  - 自己回帰的な文字生成
  - エンコーダー特徴との注意機構
  - キャッシュドデコーディング

### 2.5 位置エンコーディング

- **種類**: 学習可能な位置埋め込み
- **目的**: 空間情報の保持
- **実装**: 
  - エンコーダー：空間位置の埋め込み
  - デコーダー：シーケンス位置の埋め込み

## 3. 学習と推論

### 3.1 学習時の処理

1. **エンコーダー処理**:
   ```
   入力画像 → ResNet50 → FPN → 特徴統合 → Transformerエンコーダー
   ```

2. **デコーダー処理（教師強制）**:
   ```
   正解文字列 → 埋め込み → 位置エンコーディング → Transformerデコーダー
   ```

3. **損失計算**:
   - CrossEntropyLoss with Label Smoothing (ε=0.1)

### 3.2 推論時の処理

1. **エンコーダー処理**:
   - 入力画像から特徴抽出（学習時と同様）

2. **デコーダー処理（自己回帰）**:
   ```
   開始トークン → デコード → 次文字予測 → 入力に追加 → デコード → ...
   ```

3. **終了条件**:
   - EOSトークンの生成
   - 最大長に到達

## 4. 実装の工夫

### 4.1 効率化

- **メモリ効率**:
  - 特徴マップの次元削減 (FPNで256チャネルに統一)
  - 混合精度学習の採用
  - 勾配チェックポイント

- **計算効率**:
  - キャッシュドデコーディング
  - バッチ処理の最適化
  - デコーダーの並列化

### 4.2 品質向上

- **データ拡張**:
  ```python
  transforms = [
      RandomRotation(±10°),
      RandomResizedCrop(224×224),
      RandomAffine(translate=±10%),
      ColorJitter(brightness=0.2, contrast=0.2)
  ]
  ```

- **正則化**:
  - ドロップアウト (rate=0.1)
  - レイヤーノーマライゼーション
  - ラベルスムージング (ε=0.1)

### 4.3 最適化設定

```python
optimizer = AdamW(
    lr=0.0001,
    weight_decay=0.0001
)

scheduler = CosineAnnealingLR(
    T_max=100,
    eta_min=0.000001
)
```

## 5. メモリ要件と計算コスト

- GPU VRAM要件: 最低12GB推奨
- バッチサイズ32での処理時間:
  - 訓練: 約2.5時間/エポック（A100）
  - 推論: 約50ms/画像
- モデルパラメータ数: 約45M
