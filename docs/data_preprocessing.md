# データ前処理の詳細

## 概要

くずし字データセットの前処理パイプラインは以下の手順で実行されます：

1. データの読み込みと検証
2. 画像の前処理（リサイズ、正規化）
3. データセットの分割（訓練/検証/テスト）
4. 前処理済みデータの保存

## 前処理の実行

```bash
python scripts/preprocess_data.py \
    --config configs/model/kuzushiji_recognizer.yaml \
    --data-dir data/raw/dataset \
    --output-dir data
```

### 引数の説明

- `--config`: モデル設定ファイルのパス
- `--data-dir`: 生データセットのディレクトリパス
- `--output-dir`: 前処理済みデータの出力ディレクトリ

## 出力ディレクトリ構造

```
data/
├── processed/
│   ├── images/
│   │   ├── train/     # 訓練用画像
│   │   ├── val/       # 検証用画像
│   │   └── test/      # テスト用画像
│   └── char_mapping.json  # 文字とインデックスのマッピング
├── splits/
│   ├── train.json    # 訓練データのメタデータ
│   ├── val.json      # 検証データのメタデータ
│   └── test.json     # テストデータのメタデータ
└── preprocess.log    # 前処理のログ
```

## 前処理の詳細

### 1. データの読み込みと検証

- 文書IDごとのディレクトリをスキャン
- 座標情報ファイルの存在確認
- 文字画像の収集とラベル付け
- 文字とインデックスのマッピング作成

### 2. 画像の前処理

- リサイズ: 224×224ピクセル（設定ファイルで変更可能）
- RGB形式への変換
- 正規化:
  ```python
  mean = [0.485, 0.456, 0.406]
  std = [0.229, 0.224, 0.225]
  image = (image - mean) / std
  ```

### 3. データセットの分割

- 文書IDレベルでの分割（同じ文書が異なる分割に入らないように）
- デフォルトの分割比率:
  - 訓練: 80%
  - 検証: 10%
  - テスト: 10%

### 4. メタデータの保存

各分割（train/val/test）のメタデータJSONファイルには以下の情報が含まれます：

```json
{
  "processed_image_path": "path/to/processed/image.npy",
  "original_image_path": "path/to/original/image.jpg",
  "char": "文字",
  "label": 0,  // 数値インデックス
  "doc_id": "文書ID"
}
```

## 文字マッピング

`char_mapping.json`には以下の情報が含まれます：

```json
{
  "char_to_idx": {
    "文字": 0,
    ...
  },
  "idx_to_char": {
    "0": "文字",
    ...
  }
}
```

## ログ出力

前処理の進捗とエラーは`preprocess.log`に記録されます：

- データ読み込みの統計
- 分割サイズの情報
- エラーや警告メッセージ
- 処理完了の確認

## エラー処理

- 不正なデータ構造の検出
- 破損画像のスキップ
- 処理失敗時のエラーログ
- クリーンアップ処理

- 必要に応じたメモリ解放
