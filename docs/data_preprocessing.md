# データ前処理の詳細

## 1. 概要

くずし字データセットの前処理パイプラインは、メモリ効率と処理速度を最適化して以下の手順で実行されます：

1. マルチプロセスによるデータ読み込み
2. バッチ処理によるLMDBデータベース作成
3. 画像サイズの最適化
4. データセットの分割（訓練/検証/テスト）

## 2. 前処理の実行

```bash
# 基本的な実行
python scripts/preprocess_data.py \
    --config configs/model/kuzushiji_recognizer.yaml \
    --data-dir data/raw \
    --output-dir data

# メモリ使用量を抑制した実行
python scripts/preprocess_data.py \
    --config configs/model/kuzushiji_recognizer.yaml \
    --data-dir data/raw \
    --output-dir data \
    --max-image-size 1024 \  # 画像サイズの制限
    --batch-size 50          # バッチサイズの調整
```

### 引数の説明

- `--config`: モデル設定ファイルのパス
- `--data-dir`: 生データセットのディレクトリパス
- `--output-dir`: 前処理済みデータの出力ディレクトリ
- `--max-image-size`: 画像の最大サイズ（オプション）
- `--batch-size`: LMDBへの書き込みバッチサイズ（オプション）

## 3. パフォーマンス最適化

### 3.1 マルチプロセス処理
- CPU コアを効率的に活用
- 文書単位での並列処理
- 進捗状況のリアルタイム表示

### 3.2 バッチ処理
- メモリ使用量の制御
- LDBMへの効率的な書き込み
- 定期的なメモリ解放

### 3.3 画像サイズの最適化
- アスペクト比を保持したリサイズ
- メモリ使用量の削減
- 処理速度の向上

## 4. メモリ管理

### 4.1 メモリ使用量の削減
```python
# 画像の最大サイズを制限
max_image_size = 1024  # より小さいサイズを指定

# バッチサイズの調整
batch_size = 50  # より小さいバッチサイズを使用

# メモリ解放の定期的な実行
del image  # 不要なデータの即時解放
```

### 4.2 バッチ処理の設定
- 適切なバッチサイズの選択
- メモリ使用量のモニタリング
- 段階的なデータ処理

## 5. 処理の流れ

1. **データの読み込み（並列処理）**
   - 各プロセスが独立して文書を処理
   - メモリ効率の良いデータ構造を使用
   - 進捗状況の表示

2. **画像の前処理**
   - サイズ制限による最適化
   - 効率的なメモリ管理
   - バッチ処理の適用

3. **LMDBの作成**
   - バッチ単位での書き込み
   - メモリ使用量の制御
   - 進捗状況のモニタリング

4. **メタデータの保存**
   - JSON形式での効率的な保存
   - 必要な情報のみを保持
   - ディスク容量の最適化

## 6. リソース要件

### 6.1 最小要件
- RAM: 8GB
- CPU: 4コア
- ストレージ: データセットサイズの3倍

### 6.2 推奨要件
- RAM: 16GB以上
- CPU: 8コア以上
- SSD: 高速なストレージアクセス

## 7. トラブルシューティング

### メモリ不足の場合
1. `--max-image-size`を小さく設定（例：1024）
2. `--batch-size`を減らす（例：50）
3. 不要なプロセスを終了

### 処理が遅い場合
1. CPUコア数を確認
2. ストレージの種類を確認（SSDを推奨）
3. バッチサイズを調整

## 8. 注意事項

- 十分なディスク容量を確保すること
- 処理中はシステムリソースを占有
- バックアップを推奨
